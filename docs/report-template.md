# Report Template

建议最终报告默认包含以下头部信息。

## 视频概览

- 输入来源：`local_file | remote_url`
- 原始输入：`<source>`
- 解析视频路径：`<video_path>`
- 来源平台：`<source_host>`
- 来源标题：`<source_title>`
- 来源 ID：`<source_id>`
- Run 名：`<suggested_run_name>`
- 分析目录：`<run_directory>`
- 时长：`<duration>`
- 分辨率：`<resolution>`

## 整段总结

一句话说明视频主旨。

## 时间线

按时间段列出核心内容。

## 重点提取

提取最值得记住的配置、命令、日志、交付物、风险点。

## 风险与问题

列出：
- 敏感信息暴露风险
- 转写不确定性
- 平台下载不稳定因素

## 可执行建议

给出用户后续如何复现、如何继续分析、如何增强能力。

## 方法局限

明确说明：
- 是否是自动转写
- 是否经过人工校对
- 画面分析覆盖程度
- 对平台页下载是否为 best-effort
